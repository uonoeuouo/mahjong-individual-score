# 🏢 運用担当者向けガイド

麻雀スコア自動集計Botの日常運用マニュアルです。

---

## 📋 概要

このBotは、Discord のチャットに投稿された麻雀スコアを読み取り、Google スプレッドシートに自動集計するシステムです。

**主な機能:**
- スコア投稿の自動読み込み
- 得点の自動集計と保存
- 名前の表記揺れ吸収（ひらがな変換等）
- 各種スタッツの自動計算（順位、チョンボ数など）
- 動作ログ記録（トラブル対応時に便利）

---

## 🚀 日常の運用手順

### 1️⃣ Bot の起動

#### **起動方法 1: VSCode を使う場合（推奨）**
1. プロジェクトフォルダを VSCode で開く
2. `main.py` ファイルを開く
3. 右上の再生ボタン（▶️）をクリック

```
✅ 起動成功時のターミナル出力例:
Logged in as 個人戦スコア集計Bot#4196
```

#### **起動方法 2: ターミナルから実行**
```bash
# ターミナルをプロジェクトフォルダで開く
cd /path/to/mahjong-individual-score

# 仮想環境を有効化
source venv/bin/activate

# Bot を起動
python main.py
```

#### **🔴 起動後の確認ポイント**
- Discord アプリで該当の Bot がオンライン状態になっているか確認
- 対象チャンネルが設定されているか確認（詳しくは管理者に確認）

---

### 2️⃣ スコアの集計

**集計実行手順:**

1. 対象の Discord チャンネルで、スコア投稿が終わったことを確認
2. チャンネルに以下のコマンドを入力
   ```
   !collect
   ```
3. Bot が「🔄 ログの読み込みと集計を開始します...」と応答
4. 集計完了まで待機（通常 10～30 秒程度）
5. ✅ マークと完了メッセージが投稿される

**✅ 集計成功時の出力例:**
```
✅ 集計完了しました。スプレッドシートを更新しました。
[スプレッドシートへのリンク]
```

---

### 3️⃣ Bot の停止

**方法 1: VSCode の場合**
- ターミナルウィンドウの右側にある 🗑️ アイコンをクリック

**方法 2: ターミナルの場合**
- ターミナルで `Ctrl + C` キーを押す

**✅ 停止完了時:**
```
disconnected from Discord
```

## 📊 スプレッドシートの確認・メンテナンス

### スプレッドシート構成

| シート名 | 用途 |
|---------|------|
| **HowToUse** | スプレッドシートの簡単な説明 |
| **RawData** | スコア投稿の生データを記録 |
| **Members** | 選手名簿 |
| **Stats** | 総合統計情報（勝率、順位数、着順分布など） |
| **HeadToHead** | 直対成績を調べる |
| **YYYYMMDD** 形式 | 日別の詳細データ（新規集計時に自動作成） |

### 日常確認ポイント
- ✅ RawData シートに最新のスコアが記録されているか
- ✅ 日別シート（YYYYMMDDフォーマット）が作成されているか

## ❌ トラブルシューティング

### 症状別対応ガイド

#### 🔴 **Error: <Response [404]>**
**原因:** スプレッドシートのIDが間違っているか、サービスアカウントが招待されていない

**対応:**
1. 管理者に連絡し、`.env` ファイルの `SPREADSHEET_KEY` が正しいか確認
2. Google スプレッドシートのアクセス権限を確認
3. Bot を再起動

---

#### 🔴 **合計エラー（Total error）**
**原因:** スコアの合計が「0」になっていない投稿がある

**対応:**
1. ユーザーに投稿フォーマットを確認させる
2. 投稿例：
   ```
   A さん 54.3
   B さん 12.1
   C さん -15.5
   D さん -50.9
   ```
   合計 = 54.3 + 12.1 - 15.5 - 50.9 = **0** ✅
3. ユーザーが投稿を修正（削除→再投稿）した後、`!collect` を再実行

---

#### 🔴 **Bot がコマンドに反応しない**
**原因:** Bot がオンラインになっていない可能性

**対応:**
1. Discord アプリで Bot のステータスを確認
2. オンラインでない場合は Bot を再起動
3. 起動後 30 秒待機してから `!collect` コマンドを実行

---

#### 🔴 **スプレッドシートが更新されない**
**原因:** Google API の権限エラーまたはネットワーク接続エラー

**対応:**
1. インターネット接続を確認
2. 管理者に連絡し、`credentials.json` ファイルが正しいか確認
3. Bot を再起動
4. 再度集計コマンドを実行

---

### ログ確認方法

**ターミナルにエラーが表示される場合:**

1. ターミナルウィンドウをスクロールアップして、エラーメッセージ全体を確認
2. エラー内容をコピーして管理者に報告
3. 形式の例:
   ```
   Traceback (most recent call last):
     File "main.py", line 35, in <module>
       client.run(TOKEN)
   Error: Invalid token
   ```
